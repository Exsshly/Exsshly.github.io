yolov5备查

Date: 2023-09-11  20:39:09

致谢：Qlittle

前置准备

安装python（这里要勾选添加到Path）

下载yolov5代码，解压到工作目录

cmd或者终端cd到yolov源码目录下，准备创建虚拟环境

（其实虚拟环境在哪里都无所谓，甚至可以只有一个虚拟环境，也就是把所有需要环境的包都pip到那里，目前我是这样做的）

创建虚拟环境

上一步已经cd至代码目录下（如之前所说，可以装在别的地方，这里仍装在代码下演示），执行

python -m venv yolovenv

以创建虚拟环境，其中最后yolovenv为虚拟环境名称，可自定义

然后执行

./Scripts/activate

以激活虚拟环境，成功后终端每行最前面应出现括号及括号内自定义的虚拟环境名

安装pytorch

进入网站https://pytorch.org/

按照实际情况选择，最后在终端执行网站给出的指令以安装pytorch

然后退回至代码根目录，执行

pip install -r .\requirements.txt

安装其他依赖(若是其他包管理方式则不一定是pip)

至此环境安装完成

测试

这里使用VsCode进行测试

更改解释器，打开IDE，打开源码文件夹，右下角更改解释器，这时上面会出现刚才自己创造的虚拟环境，选择即可。运行左侧detect.py，若有错误，根据报错信息修改；若正常运行，最后会提示结果保存在~/runs/detect/exp文件夹中，打开文件夹出现图片就算成功。

说明

以上安装基于CPU，而未安装带有GPU设备的版本，GPU版本的安装较之复杂一些（无非也就是Cuda的选择和安装以及cudnn的文件拷贝），关于显卡驱动、cuda以及其他相关加速程序这里没有提及一点，特此说明。

可以通过改变代码来直接执行，这里选择传参执行而不去改动原代码
参数有很多，可以看官方文档或代码中的相应字段，这里只列出目前使用到的参数

训练

YOUR_PATH要换为自己的相应目录，下同

上述即：python train.py --weights yolov5s.pt --cfg YOUR_PATH\yolov5s.yaml --data YOUR_PATH\data.yaml --batch-size 8 --epoch 50

检测

上述即：python detect.py --weights YOUR_PATH\runs\train\exp(n)\weights\best.pt --source YOUR_PATH

断点训练

用以解决由于突发情况训练过程突然中断，而重新训练费时的问题

  1. 在train.py文件中找到函数parse_opt，修改-–resume的default参数为Ture
  2. 确认上一次运行的exp(n)文件夹中有last.pt这个最后的权重文件，重新使用指令加载即可
  3. 以下指令中的权重文件不再是yolov5s.pt，而应换成last.pt，其余保持不变。
    python train.py --weights YOUR_PATH/last.pt --cfg YOUR_PATH/yolov5s.yaml --data YOUR_PATH/data.yaml --batch-size 8 --epoch 50

导出onnx

为了可跨平台使用权重，这里导出为onnx格式的模型文件，也是在yolo自带的代码文件中，如果之前没有onnx包，可能要pip一下

pip install onnx

然后执行

python export.py --include torchscript onnx --weights YOUR_PATH/best.pt

这里的pt文件由你指定的pytorch权重，转换的onnx在同目录下
